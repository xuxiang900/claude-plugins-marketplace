# GitLab Issue 模板

> 用于创建标准化的 GitLab Issue，确保任务定义清晰、可执行。

## 使用说明

1. 复制下方模板内容到 GitLab Issue 描述
2. 填写各字段，删除不适用的部分
3. 设置正确的 Labels（任务类型 + Feature）
4. 关联到对应的 Milestone

---

## Issue 模板

```markdown
## 背景/目标

[描述任务的背景和要达成的目标]

## 验收标准

- [ ] [具体可验证的标准 1]
- [ ] [具体可验证的标准 2]
- [ ] [具体可验证的标准 3]

## 技术方案

[简述技术实现方案，复杂任务可引用设计文档]

- 设计文档：[链接]
- 参考资料：[链接]

## 影响范围

- **模块**：[受影响的模块/组件]
- **风险**：[潜在风险和缓解措施]

## 关联信息

- **设计文档**：[链接]
- **相关 Issue/MR**：[链接]
- **依赖任务**：[链接]

## AI Coding 执行约束（可选）

> 本节用于明确 AI 辅助开发时的边界，人工执行也可参考

- **允许修改的文件**：[文件范围]
- **禁止修改的文件**：[文件列表]
- **必须通过的测试**：[测试要求]
- **代码风格要求**：[风格要求]
```

## 任务类型特定模板

### type::prototype（原型验证）

```markdown
## 背景/目标

[描述需要验证的技术方案或假设]

## 验收标准

- [ ] 可行性结论明确（可行/不可行/有条件可行）
- [ ] 技术评估报告完成
- [ ] 下一步行动建议明确

## 验证范围

- **验证点 1**：[描述]
- **验证点 2**：[描述]

## 预期产出

- [ ] 可行性报告
- [ ] POC 代码（如适用）
- [ ] 技术选型建议

## 影响范围

- **模块**：[受影响的模块]
- **时间预估**：[预估时间]

## 关联信息

- **设计文档**：[链接]
- **相关调研**：[链接]
```

### type::feature（功能开发）

```markdown
## 背景/目标

[描述功能需求和业务价值]

## 验收标准

- [ ] 功能实现完成
- [ ] 单元测试通过（覆盖率 > X%）
- [ ] 集成测试通过
- [ ] 相关文档已更新

## 技术方案

[简述实现方案]

- 设计文档：[链接]

## 影响范围

- **模块**：[受影响的模块]
- **API 变更**：[是/否，如是请说明]
- **数据库变更**：[是/否，如是请说明]

## 关联信息

- **设计文档**：[链接]
- **UI 设计**：[链接]
- **相关 Issue**：[链接]

## AI Coding 执行约束

- **允许修改的文件**：`src/features/xxx/**`
- **必须通过的测试**：`npm test -- --coverage`
```

### type::bugfix（Bug 修复）

```markdown
## 问题描述

[描述 Bug 现象和影响]

## 复现步骤

1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

## 期望行为

[描述正确的行为]

## 验收标准

- [ ] Bug 已修复
- [ ] 回归测试通过
- [ ] 根因分析已记录

## 根因分析

[分析 Bug 产生的根本原因]

## 影响范围

- **模块**：[受影响的模块]
- **用户影响**：[影响范围描述]

## 关联信息

- **错误日志**：[链接或内容]
- **相关 Issue**：[链接]
```

## 测试环境

## 关联信息

- **测试计划**：[链接]

## 影响评估

- **缓解措施**：[措施]

## 验证方法

[描述如何验证变更成功]

## 关联信息

- **技术债务记录**：[链接]
- **相关文档**：[链接]

### 必须设置的 Labels

| Label 类型 | 格式 | 示例 |
|-----------|------|------|
| 任务类型 | `type::xxx` | `type::feature` |

|-----------|------|------|
| 优先级 | `P0-critical` ~ `P3-low` | `P1-high` |
| 状态 | `status::xxx` | `status::in-progress` |
| Agent | `agent::xxx` | `agent::eligible` |
